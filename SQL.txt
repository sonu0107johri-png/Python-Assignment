Database - Collection of tables - Schema

Database vs Schema

Database - collection

Schema - A database schema is the blueprint or formal 
structure of a database. It defines how data is organized,
how tables are structured, and how they relate to one another.
While the schema acts as the skeleton, it does not actually
contain any data; the data present at a specific point in time 
is instead called a database instance.

tables - collection of rows and columns
row - tuple - record
column - fields

SQL commands are categorized into different "language"  based on whether    
they affect the database's structure, the actual data or 
DDL - Data Definition Language- Create, Alter, Drop, Truncate, Rename

DML- Data Modification Language - Insert, Update, Delete, Call, Explain Call

DCL - Data Control Language - Grant, Revoke

TCL - Transaction Contro3 Language - Commit, Rollback, Savepoint

A - Atomicity

C - Consistency

I - Isolation

D - Durability

DQL - Data Query Language - Select

TQL - Transact Query Language

Functions and Operator:

Functions 

Aggregate Functions
COUNT() - Return the number of rows that match a specific condition.
COUNT(*) - Include all rows, even those with Null values.
COUNT(column) - Only counts rows where the column value is not Null
SUM() - Calculates the total sum of values in a numeric column, ignoring Null values.
AVG() - Computes the average (mean)
